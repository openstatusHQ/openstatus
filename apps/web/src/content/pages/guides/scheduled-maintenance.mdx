Use this template for planned maintenance, infrastructure upgrades, or scheduled downtime. Proactive communication builds trust and reduces support burden.

## When to Use This Template

- Database upgrades
- Infrastructure migrations
- Security patch deployments
- Third-party provider maintenance
- Performance optimization work

## Template Messages

### Scheduled (Advance Notice)

We have scheduled maintenance for [System Name] on [Date] at [Time] [Timezone].

**Expected Duration**: [Duration]
**Expected Impact**: [Impact Description]

We will provide updates throughout the maintenance window.

### In Progress

Scheduled maintenance is currently in progress. We expect to complete within the planned window.

**Current Status**: [Brief status]
**Expected Completion**: [Time]

### Completed

The scheduled maintenance has been completed successfully. All systems are operating normally.

Thank you for your patience during this maintenance window.

## Real-World Examples

### Stripe: Payment Method Maintenance

**Context**: Scheduled maintenance for TWINT payment method
**Approach**: Clear scheduling with transparency disclaimers

**Sample messaging**:
```
TWINT has an upcoming scheduled maintenance.

Note: Stripe retrieves payment method planned maintenance information
from payment method partners and the information may not be complete,
accurate, or up to date.
```

**What worked**:
- Clear about the service affected (TWINT)
- Set expectations about information accuracy
- Consistent disclaimer across all maintenance notices

**Multi-stage updates**:
1. **Scheduled**: "TWINT has upcoming scheduled maintenance on [date]"
2. **In Progress**: "Maintenance is currently in progress"
3. **Completed**: "The scheduled maintenance has been completed"

### Notion: Planned Maintenance Communication

**Approach**: Clear timing with minimal service impact

**Sample structure**:
```
Scheduled Maintenance
Time: Jan 20, 2026 | 2:00 AM - 4:00 AM PST
Impact: Brief connectivity interruptions expected
Status: Scheduled

We'll be performing database maintenance during this window.
Most features will remain available, but you may experience
brief disruptions.
```

## Advance Notice Best Practices

### Minimum Notice Periods

- **Minor maintenance** (< 30 min, no downtime): 48 hours notice
- **Standard maintenance** (1-4 hours, read-only): 1 week notice
- **Major maintenance** (> 4 hours, full downtime): 2+ weeks notice

### What to Include

‚úÖ **Always include**:
- Exact date and time (with timezone)
- Expected duration
- Impact description
- What users can/can't do
- Who to contact with questions

‚úÖ **Nice to have**:
- Reason for maintenance
- Expected benefits
- Calendar file (.ics)
- Status page link
- Estimated completion time

‚ùå **Avoid**:
- Vague timing ("sometime next week")
- Unclear impact ("may affect some users")
- No duration estimate
- Technical jargon without explanation

## Impact Level Communication

### No Downtime
```
Maintenance Window: January 25, 02:00-04:00 UTC
Impact: None - We'll be performing rolling updates with no service interruption
Action Required: None
```

### Read-Only Mode
```
Maintenance Window: January 25, 02:00-04:00 UTC
Impact: Service will be in read-only mode
What you CAN do: View all data, run reports, access historical information
What you CAN'T do: Create, update, or delete records
Action Required: Complete any urgent changes before maintenance
```

### Full Downtime
```
Maintenance Window: January 25, 02:00-04:00 UTC
Impact: Service will be completely unavailable
What to expect: You won't be able to access the service at all
Action Required: Plan accordingly, download any needed data beforehand
Alternative: [If you have a status page or alternative access method]
```

### Partial Service Impact
```
Maintenance Window: January 25, 02:00-04:00 UTC
Impact: Partial service degradation

Available:
‚úì Login and authentication
‚úì Data viewing
‚úì API read endpoints

Unavailable:
‚úó File uploads
‚úó Report generation
‚úó Data exports
‚úó Webhook delivery
```

## Timezone Considerations

Always include multiple timezones for global audiences:

```
Maintenance scheduled for:
- 02:00 UTC
- 6:00 PM PST (Jan 24)
- 9:00 PM EST (Jan 24)
- 11:00 AM JST (Jan 25)

Check your timezone: [worldtimebuddy.com link]
```

Or use relative time:
```
Maintenance begins in 24 hours.
Check your local time: January 25, 02:00 UTC
```

## FAQ to Include

Consider adding to your maintenance notice:

**Q: Can I use the service during maintenance?**
A: The service will be in read-only mode. You can view data but not make changes.

**Q: What if I'm in the middle of something?**
A: We recommend saving your work before maintenance begins. Any unsaved changes may be lost.

**Q: Will my data be safe?**
A: Yes, we perform full backups before all maintenance. Your data is secure.

**Q: What if maintenance takes longer?**
A: We'll provide updates every hour and notify you of any time extensions.

**Q: Can you reschedule?**
A: This timing was chosen for minimal impact. Please contact support if you have urgent concerns.
