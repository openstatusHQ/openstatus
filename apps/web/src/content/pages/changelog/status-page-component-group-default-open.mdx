You can now control which component groups are expanded by default when visitors land on your status page.

Previously, only the first group was auto-expanded. Now, each group has an "Open by default" toggle in the dashboard so you can choose exactly which groups your visitors see first.

- **Per-group control** - toggle "Open by default" for any component group in the status page editor
- **Multiple groups** - expand as many or as few groups as you want
- **API support** - set `default_open` when creating or updating component groups via the API
