API keys now carry a `read` or `write` scope. Read-only keys can list and inspect resources but cannot mutate anything — useful for dashboards, read replicas, and AI agents you don't fully trust.

Pick a scope when creating a key in `Settings → API Keys`. Existing keys keep full access; rotate them to scope down.
